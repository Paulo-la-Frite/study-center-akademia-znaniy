# Проектная документация на модуль капчи-пазла

## 1. Функциональное назначение

Модуль капчи-пазла предназначен для защиты формы авторизации от автоматизированных попыток подбора паролей (брутфорс-атак). Модуль реализует интерактивную проверку «человек или робот» путём сборки изображения из четырёх перемешанных фрагментов.

Пользователь должен расставить фрагменты в правильные позиции сетки 2×2. При верной сборке поля логина и пароля активируются, и пользователь получает возможность авторизоваться.

## 2. Используемые методы и параметры

| Метод | Назначение | Параметры |
|-------|------------|-----------|
| `CaptchaWindow.__init__()` | Инициализация окна капчи | `window` — окно Toplevel, `login_window` — родительское окно, `fail_callback` — функция при ошибке, `success_callback` — функция при успехе |
| `build_ui()` | Построение интерфейса (сетка 2×2, фрагменты) | Без параметров |
| `select_piece(piece_num, label)` | Выбор фрагмента из панели | `piece_num` — номер фрагмента (1-4), `label` — виджет Label |
| `place_piece(row, col)` | Вставка выбранного фрагмента в ячейку | `row` — строка (0-1), `col` — столбец (0-1) |
| `check_puzzle()` | Автоматическая проверка после заполнения всех ячеек | Без параметров |

**Параметры настройки:**
- Размер фрагмента: 117×117 пикселей (subsample=6 от исходного 702×702)
- Количество фрагментов: 4 (сетка 2×2)
- Максимальное число попыток: 3 (MAX_FAILED_ATTEMPTS в config.py)
- Порядок фрагментов при каждом открытии: случайный (random.shuffle)

## 3. Алгоритм авторизации

1. Пользователь открывает окно авторизации. Поля «Логин» и «Пароль» неактивны, кнопка «Войти» заблокирована.
2. Пользователь нажимает кнопку «Пройти проверку».
3. Открывается окно капчи с 4 перемешанными фрагментами и сеткой 2×2.
4. Пользователь кликает на фрагмент, затем на нужную ячейку. Фрагмент вставляется в ячейку.
5. После заполнения всех 4 ячеек происходит автоматическая проверка.
6. Если пазл собран верно:
   - Окно капчи закрывается.
   - В основном окне чекбокс «Я не робот» отмечается.
   - Поля «Логин» и «Пароль» активируются.
   - Кнопка «Войти» становится доступной (зелёный цвет).
7. Если пазл собран неверно:
   - Выводится сообщение «Пазл собран неверно. Попробуйте ещё раз.»
   - Окно капчи закрывается, счётчик попыток увеличивается.
   - Пользователь может попробовать снова, нажав «Пройти проверку».
8. После 3 неверных попыток:
   - Выводится сообщение «Слишком много попыток. Попробуйте позже.»
   - Кнопка «Пройти проверку» блокируется.
   - Разблокировка возможна через панель администратора (кнопка «Снять блокировку»).
9. После успешной капчи пользователь вводит логин и пароль, нажимает «Войти».
10. При неверном логине/пароле выводится сообщение «Вы ввели неверный логин или пароль.»
11. При трёх неверных попытках входа (неверный пароль) учётная запись блокируется.
12. При успешной авторизации выводится сообщение «Вы успешно авторизовались», открывается дашборд согласно роли.
